## 🟢 리스크 & 완화

| 리스크 | 영향 | 완화 |
|---|---|---|
| LLM 동시 호출 폭주 | 사내 추론 서버 과부하 | Redis 동시성 게이트(구현). 사용자별 쿼터·`llm_jobs` 사용량 적재는 확장 |
| 스케줄러 중복 실행 | 메일 중복 발송 | worker 단일 인스턴스에서 디스패치, 실행 후 `next_run_at` 재계산 |
| 잘못된 자동 발송 | 대외 신뢰 손상 | 필수 데이터 누락 시 발송 차단·담당자 알림, 드라이런 지원 |
| 단일 서버 SPOF | 전체 중단 | 백업/헬스체크, 이후 이중화 |
| 비밀 평문 노출 | 계정 탈취 | secret 앱단 암호화(Fernet) 저장, 로그 마스킹, `.env` 커밋 금지 |
| 멀티테넌트 데이터 누수 | 정보 유출 | 모든 쿼리 `agent_id`/소유자 스코프 강제, 필요 시 RLS |
| 타임존 오프셋 | 발송일 오류 | 스케줄은 Asia/Seoul 기준으로 해석, 매처 단위 테스트로 검증 |

## 범위 밖(초기)
- 모바일 네이티브 앱, 사외 접근, 에이전트 간 연계/워크플로 체이닝, 세밀한 RBAC(부서별 권한)는 초기 범위 밖 → 확장 여지로만 설계에 반영.

- 조직 SSO(OIDC) 연동, LLM 사용량 쿼터·과금
